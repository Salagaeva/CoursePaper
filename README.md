##  О проекте

Веб-приложение для учета успеваемости студентов колледжа с разделением ролей пользователей.

Система реализует работу с оценками, пользователями и учебными данными через удобные панели управления.

---

##  Цель проекта

Создание информационной системы, обеспечивающей:

* автоматизацию учета успеваемости студентов
* контроль и анализ оценок
* разграничение прав доступа пользователей

---

## Роли пользователей и функциональность системы

### Панель администратора

`admin/admin_panel.php`

Администратор управляет всей системой и ее структурой.

**Функциональность:**

* добавление пользователей (студенты, преподаватели, администраторы)
* назначение групп студентам
* назначение предметов преподавателям
* удаление пользователей
* просмотр списка пользователей

**Преимущества:**

* централизованное управление
* динамическое отображение полей в зависимости от роли
* контроль уникальности логинов

---

### Панель преподавателя

`teacher/teacher_panel.php`

Панель предназначена для работы с оценками студентов.

**Функциональность:**

* добавление оценок
* редактирование и удаление оценок
* фильтрация по предметам и датам
* просмотр среднего балла

**Особенности:**

* преподаватель может работать только со своими предметами
* все действия ограничены его правами

**Преимущества:**

* удобная фильтрация данных
* контроль успеваемости
* снижение вероятности ошибок

---

### Панель студента

`student/student_panel.php`

Панель предназначена для просмотра успеваемости.

**Функциональность:**

* просмотр всех оценок

* средний балл по предметам
* просмотр истории оценок

**Визуализация:**

* цветовая индикация

**Преимущества:**

* наглядность данных
* простой интерфейс
* быстрый доступ к информации

## База данных и модель данных

В основе системы лежит реляционная база данных MySQL, обеспечивающая хранение всех сущностей и связей между ними.

Структура базы данных разработана с учетом нормализации, что позволяет избежать дублирования данных и повысить целостность системы.

---

### Основные таблицы

**`users`**  
Хранит учетные записи пользователей и используется для авторизации.

**`students`**  
Содержит информацию о студентах и связан с таблицами пользователей и групп.

**`teachers`**  
Хранит данные преподавателей и связан с таблицей пользователей.

**`subjects`**  
Справочник учебных предметов.

**`groups_college`**  
Справочник учебных групп.

**`teacher_subjects`**  
Реализует связь между преподавателями и предметами (многие-ко-многим).

**`grades`**  
Центральная таблица, содержащая оценки студентов, даты занятий и преподавателей.

**`attendance`**  
Заготовка для учета посещаемости.

---

### Логика связей

* один пользователь соответствует одной роли
* студент и преподаватель связаны с учетной записью
* преподаватель может вести несколько предметов
* студент может иметь множество оценок
* оценки связаны с предметами и преподавателями

Все связи реализованы через внешние ключи (**FOREIGN KEY**), что обеспечивает целостность данных.

---
